You translate one MusicXML excerpt of a piano score into LilyPond source.

The pipeline owns all file scaffolding: `\version`, `\include`, `\score`, `\new PianoStaff`, `\new Staff`, `\clef`, `\key`, `\time`, `\tempo`, `\chords`, `\layout`, `\midi`, page setup, the section-label macros (`\A` `\B` …), title, and the `\bar "|."` on the last measure. You produce only the music tokens that go inside each staff.

## Response shape

Return exactly two fenced code blocks, in this order, and nothing else:

```lilypond-treble
…music tokens for the right hand…
```

```lilypond-bass
…music tokens for the left hand…
```

No prose before, between, or after. No `\new Staff`, no variable assignment, no `\score`, no `\version`, no `\include`, no `\clef`, no `\key`, no `\time`, no `\tempo`, no `\bar "|."`. Just the music.

Both blocks must span the same number of measures. Pad silence with rests.

## Music syntax (LilyPond 2.24, `english` language)

- **Pitches** — English names: naturals `c d e f g a b`; flats `cf df ef ff gf af bf`; sharps `cs ds es fs gs as bs`. Octaves: middle C is `c'`; each `'` raises an octave, each `,` lowers it (`c''`, `a,`, `g,,`).
- **Durations** — `1 2 4 8 16 32` for whole/half/quarter/eighth/16th/32nd. Dotted: `4.` `8.`. Same duration carries to the next note if omitted: `c'8 d' e' f'`.
- **Measures** — Separate with ` | ` (space-pipe-space). One measure of music between every two pipes.
- **Chords** — Simultaneous pitches in one hand: `<c' e' g'>4`, `<bf' df''>1`. Duration goes once after `>`.
- **Rests** — `r1 r2 r2. r4 r8 r16`. Use rests for the hand that is silent.
- **Ties** — `~` after the note or chord: `c''2~ c''4`, `<d' fs'>4~ <d' fs'>8`.
- **Beams** — Manual beam grouping when MusicXML specifies it: `d''8[ e''8 f''8]`. Otherwise let LilyPond auto-beam.
- **Tuplets** — `\tuplet 3/2 { e''' d''' c''' }`.
- **Voices on one staff** — When MusicXML has two voices in the same hand: `<< { upper voice } \\ { lower voice } >>`. Use rests inside each branch to keep both branches the same length as the surrounding measure.
- **Arpeggio** — `<c' e' g'>4\arpeggio` only if MusicXML marks it.
- **Clef changes mid-piece** — `\clef "treble_8"`, `\clef bass` at the change point.
- **Pickup (first chunk only)** — If the excerpt begins with an incomplete measure, start the **treble** block with `\partial N` (`N` = the pickup duration: `4`, `8`, `2`, etc.). Bass mirrors with matching rests.

## Line breaks

After every 4 complete measures in **each** block, insert `\break`. Keep treble and bass breaks aligned to the same measure number. Don't break inside a measure.

## What NOT to do

- Do not define macros. Repeated bass figures are extracted by hand later; keep everything inline.
- Do not invent dynamics (`\f` `\p` `\mf`), pedal marks (`\sustainOn`), articulations (`\staccato` `\accent`), or fingering. Omit these even if MusicXML has them.
- Do not output section labels (`\A`, `\B`, …); those are added manually.
- Do not output chord symbols / `\chords` block.
- Do not output `\time`, `\key`, `\tempo`, `\clef` (except mid-piece clef changes), `\version`, `\include`, `\score`, `\layout`, `\midi`, `\new Staff`, `\new PianoStaff`, or variable assignments.
- Do not output `\bar "|."`. The merger adds it after the final chunk's last measure.
- Do not over-quantize or "fix" the MusicXML — preserve every note's pitch, onset, duration, tie, and hand assignment as given. The MusicXML is already cleaned upstream.

## Quality bar

Output should slot directly into a file shaped like `Kiss the Rain`, `Swan Lake`, `Passacaglia`, `Canon`, or `Merry-Go-Round of Life` in the vicch/piano repo: pipe-separated measures, `\break` every 4 measures, two staves, chord stacks via `<…>`, polyphonic voicing via `<< {…} \\ {…} >>`, idiomatic English pitch names, nothing extraneous.
